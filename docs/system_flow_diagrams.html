图 1：系统总览图

这张图只讲“系统怎么跑起来”：实时 AI 检测产出告警，Wazuh 摄取，Agent 旁路研判，最后生成报告。

一条主链路先看懂

先不要看所有文件。记住这 5 步：采集流量、AI 判断、Wazuh 接收、Agent 关联、LLM 出报告。
1
流量与日志输入
Zeek、Suricata、Wazuh 和 MineShark live log 提供原始证据。
2
实时 AI 检测
predict_scan.py 调用 Transformer 模型，输出 AI 告警。
3
Wazuh 摄取告警
Wazuh 监听 /var/log/ai_alerts.json，触发规则并写入 Indexer。
4
Agent 关联证据
读取 AI 告警、Wazuh、Zeek、Suricata 和 RAG 知识。
5
生成中文报告
DeepSeek 整合证据，输出 JSON 和 Markdown 研判报告。
关键运行文件

实时检测层

  • /root/mineshark_lab/mineshark_live.log：模型输入流量行。
  • /opt/mineshark_lab/ai_engine/predict_scan.py：实时推理脚本。
  • deep_mineshark_best.pt：已训练 Transformer 模型。
  • /var/log/ai_alerts.json：AI 告警输出。

Wazuh 证据层

  • Wazuh Manager :55000：查 manager 和 agents。
  • Wazuh Indexer :9200：查 wazuh-alerts-*。
  • /var/ossec/logs/alerts/alerts.json：Indexer 失败时的本地回退。

Agent 研判层

  • /opt/mineshark_agent：本分支项目目录。
  • outputs/rag/：FAISS 知识库索引。
  • outputs/reports/：最终报告。
这张图的面试讲法
架构选择
采用旁路研判，不替换原有实时检测服务。
核心价值
把 AI 风险线索和 Wazuh/Zeek/Suricata 证据放到同一份报告里。
安全边界
只读、关联、研判，不自动封禁，不写回规则。
已验证案例
AI 告警被 Wazuh 规则 100500 摄取，Agent 成功生成报告。

你只要先能复述这句话就算懂第一层：mineshark-ai.timer 产出 AI 告警，Wazuh 收进去，Agent 再读 Wazuh 和多源日志，最后让 DeepSeek 写报告。

图 2：代码文件职责图

这张图按“运行时调用顺序”分层。先看蓝色主路径，再看每层旁边的辅助文件。

从命令到报告：代码怎么接力

不要逐个背文件名。按层记：入口、配置、Agent 编排、工具、外部证据、输出。

第 1 层：命令入口

scripts/agent/run_agent_audit.py启动 Agent 研判。它本身很薄，只负责导入项目代码并调用 CLI。
scripts/rag/build_index.py构建 RAG 索引，用于先把知识库转成 FAISS。
pyproject.toml声明依赖和命令入口。默认不安装 torch，避免 Linux 轻量部署爆盘。
↓

第 2 层：配置加载

src/mineshark/config.py读取 .env，生成 RuntimeConfig，里面有 DeepSeek、DashScope、Wazuh 和日志路径。
.env真实密钥和本机路径。这个文件不提交 Git。
.env.example模板文件，告诉别人应该填哪些配置。
↓

第 3 层：Agent 编排

src/mineshark/agent/cli.py创建 LangGraph ReAct Agent，组装任务提示，调用 DeepSeek，写报告。
src/mineshark/agent/toolbox.py把每个数据源封装成工具，并记录 tool_trace，方便复盘 Agent 调过什么。
DEEPSEEK_MODEL=deepseek-chat使用非 thinking 模式，避免工具调用循环里的 reasoning_content 兼容问题。
↓

第 4 层：Agent 可以调用的工具

src/mineshark/sensors/ai_alerts.py读 /var/log/ai_alerts.json，按 IP、时间、阈值筛选 AI 告警。
src/mineshark/integrations/wazuh.py查 Wazuh Server API、Indexer API，失败时回退本地 alerts.json。
src/mineshark/sensors/logs.py读 Zeek conn.log 和 Suricata eve.json，补网络侧上下文。
src/mineshark/rag/store.py加载 FAISS 索引，按语义相似度找安全知识。
src/mineshark/rag/embeddings.py调用 DashScope 的 text-embedding-v4，生成 query 向量。
configs/reporting/security_playbook.jsonlRAG 的原始知识库，包含 C2、误报治理、规则解释等条目。
↓

第 5 层：输出与回归验证

outputs/reports/agent_audit_report.md给人看的中文研判报告。
outputs/reports/agent_audit_report.json给系统看的结构化结果，包含 tool_trace 和 Agent messages。
tests/*.py单元测试覆盖 AI 告警读取、Wazuh、RAG、Zeek/Suricata、Agent CLI。
面试时按这 3 句话讲
第一句
入口脚本只负责启动，真正逻辑在 src/mineshark。
第二句
toolbox.py 是中心，它把所有证据源变成 Agent 可调用工具。
第三句
报告不是凭空生成，而是由 AI 告警、Wazuh、Zeek、Suricata、RAG 共同支撑。
追问准备
如果问“坏了怎么排查”，先看 agent_audit_report.json 里的 tool_trace。

你可以把图二理解成“调用栈”：run_agent_audit.py → agent/cli.py → agent/toolbox.py → 各类工具文件 → outputs/reports。

图 3：一次完整研判数据流图

这张图回答：一条 MineShark AI 告警如何变成最终报告。

sequenceDiagram
    autonumber
    participant LiveLog as /root/mineshark_lab/mineshark_live.log
    participant Timer as mineshark-ai.timer
    participant Predict as predict_scan.py
    participant Model as deep_mineshark_best.pt
    participant AIFile as /var/log/ai_alerts.json
    participant Wazuh as Wazuh Manager
    participant Indexer as Wazuh Indexer
    participant AgentCLI as run_agent_audit.py
    participant Toolbox as AgentToolbox
    participant RAG as FAISS RAG
    participant DeepSeek as DeepSeek Chat
    participant Reports as outputs/reports/*.json + *.md

    Timer->>Predict: 定时启动实时 AI 扫描
    Predict->>LiveLog: 读取新增 MineShark 流量行
    Predict->>Model: 构造张量并执行 Transformer 推理
    Model-->>Predict: 返回恶意概率
    alt 概率 >= threshold
        Predict->>AIFile: 追加 JSONL 告警
        AIFile->>Wazuh: Wazuh logcollector 摄取
        Wazuh->>Indexer: 写入 wazuh-alerts-* 索引
    else 概率 < threshold
        Predict-->>Timer: 不写告警，只更新 state
    end

    AgentCLI->>Toolbox: 创建工具箱和 LangGraph tools
    AgentCLI->>DeepSeek: 发送系统提示和任务上下文
    DeepSeek->>Toolbox: 调用 query_mineshark_ai_alerts
    Toolbox->>AIFile: 读取 AI 告警
    AIFile-->>Toolbox: 返回模型风险线索
    DeepSeek->>Toolbox: 调用 query_wazuh_alerts
    Toolbox->>Indexer: 查询相关 Wazuh 告警
    Indexer-->>Toolbox: 返回 Wazuh 告警证据
    DeepSeek->>Toolbox: 调用 query_wazuh_agents
    Toolbox->>Wazuh: 查询 manager/status 与 agents
    Wazuh-->>Toolbox: 返回 Manager 和 Agent 状态
    DeepSeek->>Toolbox: 调用 query_zeek_context
    Toolbox->>Toolbox: 读取 Zeek conn.log
    DeepSeek->>Toolbox: 调用 query_suricata_alerts
    Toolbox->>Toolbox: 读取 Suricata eve.json
    DeepSeek->>Toolbox: 调用 retrieve_security_knowledge
    Toolbox->>RAG: 用 DashScope embedding 检索 FAISS
    RAG-->>Toolbox: 返回安全知识片段
    Toolbox-->>DeepSeek: 返回所有工具证据
    DeepSeek-->>AgentCLI: 生成中文研判报告正文
    AgentCLI->>Reports: 写入 JSON 报告和 Markdown 报告
